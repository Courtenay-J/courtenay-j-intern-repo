# Kanban
## How does Kanban help manage priorities and avoid overload?
Kanban boards provide a visual summary of the state of the sprint as a whole: what's been completed, what's in progress, what hasn't been started, and if there's any task dependencies or blockages. 

Placing limits on categories - e.g. how many "In Progress" items you can have at once - has a number of benefits. It prevents visual clutter, making it easier to assess tasks. It discourages the team from spreading themselves too thin, by working on too many different tasks at once rather than finishing what's been started. It also highlights bottlenecks in the project - if there are seven items in a category with a max of three, something may be preventing proper completion of these tasks.
