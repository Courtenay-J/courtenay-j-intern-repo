## Useful Shortcuts
### Navigation
- **CTRL+K, CTRL+S** - Open Keyboard Shortcuts
- **CTRL+K, F12** - Open Definition to the side
- **CTRL+SHIFT+V** - Open preview (HTML/markdown/etc.)
- **SHIFT+ALT+F12** - Find all references

### Editing
- **CTRL+Y** - Redo
- **CTRL+K, CTRL+C** - Convert line into comment, and vice-versa. (Either current line or highlighted lines)
- **CTRL+SHIFT+K** - Delete current line
- **CTRL+ALT+BACKSPACE** - Remove brackets
- **ALT+DOWNARROW or ALT+UPARROW** - Move current line up/down

### Debugging
- **F5** - Start Debugging
- **F10** - Start Debugging Process, but stop on entry

*In test mode:*
- **ALT+F8** - Go to next test failure

### Terminal Commands
- **F3** - Find next
 - **SHIFT+F3** - Find previous
- **CTRL+W** - Kill active terminal
- **CTRL+SHIFT+ALT+`** - New Terminal Window

## Reflection
### Which shortcuts do you think will be the most useful in your daily work?
- Converting line to comment: speeds up writing comments. Also allows for quick enabling/disabling of lines of code when debugging.
- Find all references: good for refactoring

(I do not have much experience with the VS Code terminal, and in the past have run test scripts via windows CMD, so I cannot gauge their usefulness just yet.)

### How can keyboard shortcuts improve your productivity?
Automates very basic, yet very common tasks. Speeds up navigation. Can be used to reconfigure the layout of VS Code into a preset relevant to your current task.

They also improve ergonomics: the less times you have to take your hands off your keyboard and put them on your mouse (or vice-versa), the less strain you put on your wrist and shoulder.
