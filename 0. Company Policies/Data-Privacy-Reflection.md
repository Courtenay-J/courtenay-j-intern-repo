# Data Privacy Reflection

## Reflection
### What steps can you take to ensure you handle data securely in your daily tasks? / How should you store, share, and dispose of sensitive information safely?
- Minimise data stored & processed: store only what is relevant, and discard/do not save anything that is not
- Store sensitive data in secure locations only
- Only store information while it is necessary for the project, and delete if/when finished
- Encrypt data to minimise who can see it (Focus Bear uses double encryption for user data)
- Anonymise data that would identify a user (e.g. name, IP address, email. use pseudonyms or id numbers if necessary, aggregate data if not)
- Do not discuss data with anyone outside of the project (and especially not outside of Focus Bear)


### What are some common mistakes that lead to data privacy issues, and how can they be avoided?
- Unrestricted access: allowing anyone (in or outside the company) to access private data, including people and systems that don't need to interact with it. Decide who the data is relevant to early, and only give access to relevant team members and processes.
- Unnecessary duplication: if data is being saved in multiple places, then it will be more work to encrypt, control access to it, delete it, etc. Keep it in one place and minimise repetitive, redundant code.
